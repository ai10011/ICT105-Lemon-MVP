# Success Metrics and Decision Rules

## 1. Success Metrics
| Metric ID | Metric | Target | How to Measure | Related Requirement |
|---|---|---:|---|---|
| M-01 | Task completion rate | 70% or higher | Completed tasks / total tasks | FR-01, FR-02 |
| M-02 | Form completion accuracy | 80% or higher | Valid submissions / total submissions | FR-03, FR-10, FR-11 |
| M-03 | Search success rate | 70% or higher | Users who find target record / total users | FR-05, FR-06 |
| M-04 | Admin task completion rate | 80% or higher | Completed admin tasks / total admin tasks | FR-08, FR-09 |
| M-05 | Dashboard comprehension | 80% or higher | Accurate interpretation of stats / total staff | FR-12 |
| M-06 | Usability rating | Average 4 out of 5 | Feedback form rating | All |
| M-07 | Feature usefulness rating | Average 4 out of 5 | Feedback form rating | MVP features |

## 2. Decision Rules
| Result | Decision |
|---|---|
| All metrics meet target | Finalize prototype design and proceed to full system development (backend integration). |
| Some metrics fail | Adjust specific UI elements (e.g., form fields, search filters, admin controls) and retest. |
| Most metrics fail | Revisit core requirements, user stories, and problem evidence before making further prototype changes. |
