# Initial IT Venture Idea Log

## Idea 1: Smart Campus Lost-and-Found System
### Problem Area
Students often lose items on campus and do not know where to report or search for them.

### Target Users
University students, security staff, and administrative staff.

### Current Alternative
Students usually post in chat groups or ask security manually.

### Proposed IT Solution
A web-based lost-and-found platform where users can report found items, search missing items, and contact the responsible office.

### Possible Technology
HTML/CSS/JS prototype, database or Google Sheet backend, image upload feature, search/filter function.

### Why This Is Suitable
The system can be prototyped using web development, database design, UI/UX design, and basic validation with real students.

## Idea 2: Digital Notice Board
### Problem Area
Scattered events informations within university.

### Target Users
Students

### Current Alternative
Line Chats in university.

### Proposed IT Solution
A simple, zero-commission community web or mobile based app informing what events are happening in university and can register within deadline.

### Possible Technology
Web application with simple catalog display, and SMS alerting API.

### Why This Is Suitable
Convinent for students and event organizer.

## Idea 3: Student Skill-Share and Project Matchmaker
### Problem Area
Students looking to build startup teams or assignments struggle to find peers outside their immediate friendship circles who have complementary technical or design skills.

### Target Users
University students looking for project collaborators.

### Current Alternative
Random assignments by professors or posting on massive, unstructured social media forums.

### Proposed IT Solution
A web dashboard where students create profiles tagging their skills (e.g., Python, Figma) and find projects matching their needs.

### Possible Technology
Web application dashboard, data search filtering components, simple match notification system.

### Why This Is Suitable
Excellent for practicing UI/UX layouts and basic database filtering operations using mock data components.
