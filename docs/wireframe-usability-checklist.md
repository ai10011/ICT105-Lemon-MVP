# Wireframe Usability Review Checklist

Use this checklist before submitting Lab 05.

| Check Item | Yes/No | Evidence or Screenshot |
|---|---|---|
| The homepage clearly states the problem and main action. | Yes | Homepage displays "What did you lose today?", search bar, and **Report an Item** button. |
| The main user pathway can be followed from start to finish. | Yes | User can search or report an item → browse results → view item details → submit/claim item. |
| The input form includes required fields and validation feedback. | Yes | Report form includes Report Type, Item Details, Category, Description, Location, Photo Upload, Privacy Policy checkbox, and Submit Report button. |
| The record list includes search/filter/category controls. | Yes | Browse Items screen includes Filters (Status, Category, Item Type, Location) and item cards. |
| The detail view shows full information and current status. | Yes | Detail page displays item image, description, location, status, and action buttons (Open Claims, Message Finder). |
| The admin view includes at least one management action. | Yes | Admin Dashboard provides **Approve Claims**, Pending Claims, and Recent Reports management. |
| The dashboard shows meaningful summary information. | Yes | Dashboard displays Total Reports, Pending Claims, Recovery Rate, and Recent Reports. |
| Navigation is consistent across screens. | Yes | All screens use the same top navigation (Home, Report, Dashboard) and consistent layout. |
| The design uses realistic sample data. | Yes | Screens contain sample item names, report statistics, claim records, and dashboard metrics. |
| Each screen maps to system-requirements.md. | Yes | Homepage (FR-01, FR-02), Report Form (FR-03, FR-10, FR-11), Browse Items (FR-05, FR-06, FR-08), Detail View (FR-07, FR-08), Admin Dashboard (FR-09), User Dashboard (FR-12). |
